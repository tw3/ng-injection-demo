"Accidental override"

The bug: A parent relies on a root-level service for shared state. A child component re-provides the same service in its own providers array. The child silently receives a fresh, empty instance instead of the root one, and state appears to be "lost."

The parent sees the root instance, but the child gets a brand-new one. Increment the parent's counter — the child's counter stays at zero:

Parent (uses root instance)

The fix: Remove the providers array from the child component. Without its own provider, the child will inherit the root instance and share state with the parent as expected.
